FE 3명, BE 2명, UI/UX 1명

::: tip 맛집을 리뷰하고 공유할 수 있는 지도형 SNS 서비스

::: important 다른 팀에서 제작해서 배포한 1.0 버전의 리팩토링 팀원으로 합류

##### 서비스 주요 기능

- 카카오 지도 API로 맛집의 위치와 정보 제공
- 음식 카테고리별 피드 모아보기
- 유저 팔로우/언팔로우
- 피드, 식당에 좋아요 표시
- 유저별 맛집 지도 제공

##### 사용 기술

- `Next.js`, `Typescript`, `Tailwind CSS`, `Zustand`, `React Query`

##### 맡은 역할

::: note 회원가입, 식당 상세정보, 맛집 지도 페이지 및 관리자 페이지 리팩토링

- **피드 카테고리 페이지 제작**

  - `useInfiniteQuery`로 스크롤 위치에 따른 데이터 호출
  - 카테고리별 query key로 캐시 데이터 활용

- **Drawer(바텀시트) 컴포넌트 제작**

  - Drawer의 내부 요소 높이 변화를 측정해 슬라이드, 상단 고정 요소, y축 반응형 등의 여러가지 기능 구현
  - 재사용성을 높이기 위해 다양한 옵션을 props로 제공

- **컴포넌트 단위 state로 관리되던 로직을 React Query로 전역에서 관리**

  - `state`와 `useEffect`로 관리되어 깜빡임이 존재하던 코드를 `useMutation` 으로 정리
  - 데이터 변이에 따른 최신 데이터 호출을 하나의 훅에서 관리할 수 있도록 추상화

- **관리자 페이지 리팩토링**
  - 유저 클라이언트와 마찬가지로 `state`로 관리되던 페이지네이션 로직을 prefetch를 활용해 사용자 경험 향상
  - 모달에 `portal`을 적용하여 부모 요소와의 결합도 낮춤
